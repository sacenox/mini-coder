Why mini-coder

mini-coder narrows the focus to one job: being a good coding agent in the terminal. Instead of owning every layer itself, it leans on proven dependencies, keeps the codebase flat and simple, treats the agent workflow as the focus, optimizes for startup and turn latency, and streams progress end-to-end.

  • Lean on proven dependencies so the core stays focused on agent work.
  • Flat, simple codebase with no internal abstraction maze.
  • Agent-first behavior centered on reading code, editing files, and verifying results.
  • Performance where startup time and turn latency matter more than feature sprawl.
  • Streaming everywhere so text, reasoning, and tool activity are visible while they happen.

Workflow

  1. Send a task. A plain-language request becomes a user message immediately.
  2. Build context. mini-coder assembles the system prompt, message history, tool definitions, current working directory, and git state.
  3. Stream the turn. Assistant text, reasoning, and tool calls update the UI incrementally instead of waiting for a full response.
  4. Run tools when needed. Tool results are appended to the same turn and the loop continues with the updated context.
  5. Stop or interrupt safely. The loop runs until the model stops. If an overlay is open, Escape closes it without touching your draft; otherwise it interrupts the turn and keeps the partial output for the next turn or /undo.

There is no hard step limit. The loop continues until the model reaches a normal stop or you interrupt it.

Built-in tools

shell

Runs shell commands in the session directory and returns stdout, stderr, and exit code. Large output is truncated as a safety guard against context blowups.

edit

Performs exact-text replacement in a single file. It fails deterministically if the target is missing or ambiguous, and it can create new files when needed.

readImage

Reads PNG, JPEG, GIF, and WebP files as model input. Only registered when the active model supports images.

Plugins can add more tools, but the core stays intentionally small: shell and edit do the real work, and readImage appears only when it is useful.

Features

Core workflow

  • Multi-provider model support — Anthropic, OpenAI, Google, Bedrock, Mistral, Groq, xAI, OpenRouter, Ollama, Copilot, and more via pi-ai.
  • Streaming TUI — markdown conversation log, tool blocks with diffs, animated divider, multi-line input, and a one-line pill status bar with independent ANSI16 effort/context tones.
  • Session persistence — SQLite-backed sessions with undo, fork, resume, and cumulative usage stats. Sessions are scoped to the working directory.
  • Reasoning and verbosity controls — toggle thinking visibility and verbose tool rendering on demand. Preferences persist across launches.

Context and extensibility

  • AGENTS.md support — project-specific instructions discovered root-to-leaf, with ~/.agents/ for global instructions.
  • Agent Skills — skill catalogs exposed in the prompt, and /skill:name injects a skill body into the next user message.
  • Plugins — optional tools, integrations, theme overrides, and prompt suffixes without bloating the core.

Interface

The UI is a three-zone terminal layout: a scrollable conversation log, a divider that animates while the agent is working, and a multi-line input with a one-line status bar.

What you see

  • User messages with a subtle background.
  • Assistant messages streamed as markdown.
  • Reasoning content that can be shown or hidden.
  • Shell, edit, and plugin tool blocks with compact previews or full output.
  • Status pills for model, effort, cwd, git state, cumulative usage, context usage, and cost.

Input and keys

  • Enter submits and Shift+Enter inserts a newline.
  • Tab autocompletes paths, or opens the command picker when the draft starts with /.
  • Ctrl+R searches global raw prompt history.
  • Escape closes the overlay without changing the draft, or interrupts the running turn when no overlay is open.
  • Ctrl+C exits cleanly.
  • Ctrl+D exits cleanly when the input is empty.
  • :q exits cleanly.
  • Ctrl+Z suspends the process.
  • Mouse wheel scrolling reviews conversation history.
  • /skill:name prepends a skill body to the next user message.

Commands

/model
Switch models and save the choice as the global default.
/session
Open the session picker for the current working directory.
/new
Start a fresh session and reset the running token and cost totals.
/fork
Fork the current chat into a new session, keep the original, and add a UI-only Forked session. note.
/undo
Remove the last conversational turn without touching filesystem changes.
/reasoning
Show or hide model thinking. The setting is saved and restored on launch.
/verbose
Expand shell output plus edit previews and edit errors in the conversation log.
/login
Sign in with a supported OAuth provider.
/logout
Remove saved OAuth credentials for a logged-in provider.
/effort
Choose low, medium, high, or xhigh reasoning effort.
/help
Show commands, current toggles, loaded AGENTS.md files, skills, and plugins.

Headless one-shot mode

mini-coder also supports a non-interactive one-shot mode for scripts and benchmark harnesses. 

$ mc -p "summarize this repo"
$ printf '%s\n' 'fix the failing tests' | mc
  • Automatic mode selection. Headless mode starts when -p/--prompt is provided or when stdin or stdout is not a TTY.
  • No interactive fallback. If stdout is redirected but stdin is still interactive, pass -p; headless mode will not fall back to an interactive prompt.
  • Shared input parsing. Plain text, /skill:name, and standalone image paths use the same parser as the TUI.
  • Raw NDJSON output. Stdout carries text deltas, tool activity, final messages, and done, error, or aborted outcomes.
  • Normal session persistence. Headless runs still persist like normal sessions and show up in /session history for that working directory.
  • No interactive slash commands. Commands such as /model, /session, and /help are not available in headless mode.

Sessions, settings, and context

mini-coder persists data under ~/.config/mini-coder/. Sessions live in mini-coder.db, and global defaults live in settings.json. Session history is stored in SQLite, prompt history is searchable across working directories, and cumulative usage is computed from assistant message usage data.

  • Sessions are cwd-aware. Resume recent work in the current project without mixing it with unrelated directories.
  • Settings are global. Model, effort, reasoning visibility, and verbose mode persist across launches.
  • Repo context is discovered. AGENTS.md files are collected root-to-leaf, with ~/.agents/ included for global instructions.
  • Skills are discoverable. Skill catalogs are exposed in the prompt, and /skill:name injects a skill directly into the next user message.
  • Plugins extend the core. They can register tools, append prompt context, and override the active theme.

The system prompt also carries the current date, working directory, and git state so the model sees the same situational context you would expect from a good shell prompt.

Built on proven parts

mini-coder uses pi-ai for provider support, streaming, tool calling, usage tracking, and OAuth, and cel-tui for the terminal UI. That keeps the core focused on agent behavior instead of reimplementing provider glue or terminal rendering.
